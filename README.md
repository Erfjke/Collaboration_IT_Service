## **Архитектура**

### C4-модель

Collaboration Service – корпоративная платформа для совместной работы и организации процессов IT-компании.

Цель проекта – создание единой среды, где сотрудники могут планировать работу, вести проекты и задачи, управлять спринтами и релизами, взаимодействовать через чаты и хранить документацию в корпоративной Wiki.

Основные возможности:
– управление проектами, задачами, спринтами и релизами;
– встроенный календарь с дедлайнами и событиями;
– Wiki-раздел с иерархией страниц и файлами;
– система чатов для личных и групповых коммуникаций;
– хранение файлов и вложений внутри проектов;
– централизованный доступ и прозрачная структура данных.

Проект реализован в архитектурном стиле Clean Architecture

Иллюстрация и описание архитектура ПС

1. Диаграмма контейнеров уровня системы (Context + Containers)

Диаграмма показывает взаимодействие сотрудника IT-компании с системой через веб-интерфейс.
Пользователь работает с UI — одностраничным приложением (React SPA), которое получает статический контент (HTML, CSS, JavaScript) и взаимодействует с базой данных (PostgreSQL) для чтения и сохранения информации.
Данный уровень отражает общую архитектуру и контуры системы — от пользователя до хранилища данных.

<img width="1659" height="1222" alt="image" src="https://github.com/user-attachments/assets/0cfb15dc-7772-4f15-9145-6931623d859f" />

2. Диаграмма контейнеров уровня сервисов (C4 Container Level)

На этом уровне система представлена как набор модулей: Auth, Project, Calendar, Users, Communication и Tasks.
Каждый сервис изолирован логически, имеет собственный слой Controller–Service–Repository и собственную базу данных (PostgreSQL).
UI взаимодействует с сервисами по REST-интерфейсу, обеспечивая модульность, расширяемость и простоту интеграции.
Диаграмма отражает логику распределения компонентов между сервисами и связи между ними.

<img width="2922" height="1371" alt="C4 component (1)-2 (4) drawio" src="https://github.com/user-attachments/assets/8e5a7edd-7c2d-46d2-8763-439ff6daa2df" />


3. Диаграмма уровня кода (C4 Code Level)

Подробно раскрывает внутреннюю структуру каждого контейнера: Controller обрабатывает запросы, Service содержит бизнес-логику, а Repository отвечает за доступ к данным.
Связи между контейнерами показывают REST-взаимодействие и проверку токенов/ролей через Auth Service.
Этот уровень демонстрирует реализацию принципов Clean Architecture и четкое разделение ответственности между слоями приложения.
